
### The Bug in `gitStyleSync.ts` (Line 309):

```typescript

```

**Why this was wrong**:
- Merge correctly detected deletion and kept tombstone
- But then immediately filtered it out!
- `saveDownloadedDataToLocal()` received data without tombstone
- `syncLocalDataToFirestore()` tried to read tombstone from local storage
- But it wasn't there anymore!
- Deletion never uploaded to cloud

## Why Tombstones Are Important

**Tombstones** are deleted items marked with `deleted: true`. They're essential for multi-device sync:

### Without Tombstones:
```
Phone A: Has Entry X v1
Phone B: Has Entry X v1

Phone A: Deletes Entry X
‚Üí Removes from local storage completely
‚Üí Sync happens
‚Üí Cloud still has Entry X v1
‚Üí Phone B syncs
‚Üí Downloads Entry X v1 (not deleted!)
‚Üí Entry comes back on Phone A! üò±
```

### With Tombstones:
```
Phone A: Has Entry X v1
Phone B: Has Entry X v1

Phone A: Deletes Entry X
‚Üí Marks as deleted: true, version: v2
‚Üí Tombstone stays in local storage
‚Üí Sync happens
‚Üí Uploads Entry X v2 (deleted: true)
‚Üí Cloud has Entry X v2 (deleted: true)

Phone B: Syncs
‚Üí Downloads Entry X v2 (deleted: true)
‚Üí Saves tombstone locally
‚Üí Display filters out deleted items
‚Üí Entry deleted on Phone B! ‚úÖ
```
